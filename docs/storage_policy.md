# Storage Policy

## Назначение
Репозиторий использует Git LFS для крупных бинарных файлов, связанных со сценами, ассетами, видео, моделями и экспортами.

## Что хранится через Git LFS
Следующие расширения обязаны идти через Git LFS:
- *.blend
- *.fbx
- *.obj
- *.stl
- *.png
- *.jpg
- *.exr
- *.mp4
- *.pt
- *.onnx
- *.zip

## Правила
1. Не коммитить крупные бинарные файлы в обычный Git.
2. Все новые файлы указанных типов должны автоматически попадать под правила из `.gitattributes`.
3. Для файлов больше 50 MiB обычный Git в этом репозитории запрещен по внутренней политике проекта.
4. Генерируемые артефакты обучения, если они не являются обязательными для версионирования в Git, должны храниться через DVC или во внешнем artifact storage.
5. Перед коммитом крупных ассетов проверять:
   - `git status`
   - `git lfs ls-files`
   - отсутствие крупных бинарников вне LFS.

## Примечание
Если крупный файл уже попал в историю Git без LFS, одного добавления правила в `.gitattributes` недостаточно. Такой файл надо переводить в LFS отдельно.
